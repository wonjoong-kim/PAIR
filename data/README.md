# Data

The eight dialog splits per dataset that the PAIR pipeline needs are
checked into the repository:

```
data/
├── gta/
│   ├── clean_train.json                    (~840 KB)
│   ├── clean_test.json                     (~250 KB)
│   ├── contaminated_train.json             (~1.0 MB)
│   ├── contaminated_test.json              (~300 KB)
│   ├── matched_clean_train.json            (~950 KB)
│   ├── matched_clean_test.json             (~280 KB)
│   ├── matched_contaminated_train.json     (~960 KB)
│   └── matched_contaminated_test.json      (~290 KB)
└── toolbench/
    ├── clean_train.json                    (~6.5 MB)
    ├── clean_test.json                     (~1.7 MB)
    ├── contaminated_train.json             (~7.3 MB)
    ├── contaminated_test.json              (~1.9 MB)
    ├── matched_clean_train.json            (~12 MB)
    ├── matched_clean_test.json             (~3.0 MB)
    ├── matched_contaminated_train.json     (~12 MB)
    └── matched_contaminated_test.json      (~3.0 MB)
```

Everything else under `data/` (features, trained probes, GRPO checkpoints)
is regenerated by the scripts in this repo and is gitignored.

## Roles of the splits

* `clean_train` / `clean_test` — the original agentic dialogs. Used as
  GRPO training and evaluation prompts in `grpo/scripts/run_single.py`
  and `grpo/scripts/evaluate.py`.
* `contaminated_train` / `contaminated_test` — the same episodes with
  one or more assistant turns rewritten to be wrong. Provides negative
  examples for PAIR probe training.
* `matched_*` — flattened per-turn records derived from the four
  episode splits above. Each record holds the dialog prefix up to one
  assistant turn plus its binary label (`1` = clean, `0` = contaminated).
  These are the splits the probes train and evaluate on.

## Where each file is consumed

| Stage | Reads |
|-------|-------|
| `probing/extract_features.py`        | all eight splits (only the ones present on disk are processed) |
| `probing/extract_multi_layer_attn.py` | same |
| `probing/train_pair.py`              | `features/.../matched_clean_train`, optionally `matched_contaminated_train` (controlled by `--train-modes`) |
| `probing/train_pair_new.py`          | same |
| `probing/eval_pair.py`               | `features/.../matched_clean_test` and `matched_contaminated_test` |
| `grpo/envs/gta_env.py`               | `gta/clean_{train,test}.json` (for GRPO rollouts) |
| `grpo/envs/toolbench_env.py`         | `toolbench/clean_{train,test}.json` |

## JSON schema

Episode-level files (`clean_*.json`, `contaminated_*.json`) are JSON
objects keyed by episode ID:

```jsonc
{
  "12": {
    "dialogs": [
      {"role": "user", "content": "..."},
      {"role": "assistant", "thought": "...", "tool_calls": [...]},
      {"role": "tool", "name": "...", "content": {"type": "text", "content": "..."}},
      {"role": "assistant", "content": "Final Answer: ..."}
    ],
    "tools": [{"name": "...", "description": "...", "inputs": [...]}],
    "files": [{"type": "image", "path": "..."}],
    "gt_answer": {"whitelist": [["..."]], "blacklist": [["..."]]},
    "contaminated_turn_indices": [3]    // contaminated_* splits only
  }
}
```

`matched_*` splits are flat lists of per-turn records:

```jsonc
[
  {
    "episode_id": "12",
    "turn_idx": 3,
    "dialogs": [...],   // full dialog prefix up to and including turn_idx
    "label": 0          // 1 = clean, 0 = contaminated
  }
]
```

## Optional: balanced indices

`train_pair.py` and `train_pair_new.py` look for

```
data/features/{model}/{dataset}/matched_clean_train/balanced_indices.npy
```

to down-sample the majority class. If the file is missing the scripts
fall back to using all rows of `matched_clean_train`.
